Week 9

Date: Febuary 28th, 2019

Total hours: 6

Description of Design Effors:

This week I spent time working with the various MSP432 timer modules. We need a module uses a capture compare value for when a timeout occurs, which will be when the Raspberry Pi doesn't send a response in time or when the keypad is left unattended. For that I plan on using the Timer32 module which is a 32-bit count down timer at 3Mhz (SMCLK) that should be fast enough and allow us to set speicific timeout values. The other timer module will be the Timer A module. This timing module offers more flexibility and has an easy configuration. Next week (after spring break) I will integrate the Timer32 module with the uart module to track how long responses are taking and make sure that characters are not being sent out too quickly.

Week 14

Date: April 12th, 2019

Total hours: 8

Description of Design Effors:

This week I spent some time digging through the MSP432 User Guide and Driver Library to find functions for verifying internal and external clock frequencies. The micro supports a 48MHz external oscillator and it might be more reliable than the internal ones. These functions also verified that the subsystem clock was operating at 24Mhz which meant our UART module configuration was correct. I also ran into problems with the keypad, for whatever reason the keypad was not reading in pin values correctly. I ended up moving some of the pin connections on the breadboard and started getting reliable presses again.

I also worked on cleaning up the code we had as it wasn't very cohesive. This involved creating new files for the LEDs and a Lock file that abstracts the functionality of toggleing on GPIO and recording a sequences of button presses. This code is much easier to maintain and debug and will hopefully be beneficial moving forward. Unfortunately we weren't able to program our PCB yet, but we should be able to figure it out in the next few days. Next week I will (hopefully) be debugging the code on our board.
